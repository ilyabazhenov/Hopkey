# Hopkey

**Увидел ключ — прыгнул в задачу.** · *Hop to the key.*

A macOS menu-bar app that spots Jira issue keys (`PROJ-123`) anywhere — in your
clipboard or selection — and jumps straight to the ticket.

Приложение для macOS в строке меню, которое распознаёт ID задач вида `PROJ-12345`
где угодно в системе и открывает соответствующую ссылку Jira
(`https://jira.example.com/browse/PROJ-12345`) в браузере по умолчанию.

Решает рутину: коллеги присылают только номер задачи — больше не нужно вручную собирать ссылку.

### Почему «Hopkey»

В Jira `PROJ-123` официально называется **issue key** — «ключ задачи». Hopkey узнаёт этот
ключ где угодно и одним прыжком (**hop**) переносит тебя прямо на тикет — без копипаста и
ручной сборки ссылок. **Hop + key = «hop to the key».** Имя обыгрывает тройной смысл «key»:
*issue key* (`PROJ-123`) → *keyboard key* (срабатывает по хоткею `⌃⌥J`) → *the key* к задаче.

> **Почему не «замена на месте»?** Превратить текст в кликабельную ссылку прямо внутри
> Telegram/Slack/Mail на macOS нельзя: системная «Замена текста» умеет только статичную замену
> строка→строка (без шаблонов), а вставлять ссылки в окна чужих приложений ОС не разрешает.
> Поэтому приложение распознаёт ID и сразу открывает нужную страницу.

## Как это работает

Два триггера с общим парсером:

1. **Авто-детект буфера обмена** (основной, без особых разрешений). Скопировали текст с
   `PROJ-12345` → появляется кликабельное уведомление. Клик открывает ссылку. В настройках
   можно выбрать «Сразу открыть в браузере» — тогда браузер открывается без уведомления.
2. **Глобальные хоткеи** (опционально, включаются в Настройках). Выделили ключ в любом
   приложении → нажали хоткей → ссылка открывается или копируется мгновенно. Хоткеи
   открыть (по умолчанию `⌃⌥J`) и скопировать (`⌃⌥K`) требуют Accessibility (синтез
   `Cmd+C` над выделением); хоткей окна ручного ввода (`⌃⌥O`) — нет.

Несколько ID в одном тексте открываются/перечисляются вместе. Регистр не важен
(`proj-7` → `PROJ-7`).

## Сборка

Нужен Xcode / Swift 5.9+ (macOS 13+).

```bash
./build.sh
```

Скрипт соберёт release-бинарник, упакует его в `build/Hopkey.app` и подпишет
стабильным self-signed-сертификатом «Hopkey Dev» (или ad-hoc, если его нет —
см. `./setup-signing.sh`).

## Разработка (watch-режим)

Авто-пересборка и перезапуск при изменении файлов в `Sources/`:

```bash
make watch
```

Использует `fswatch` (если установлен — `brew install fswatch`), иначе опрос раз в секунду.
`Ctrl+C` останавливает наблюдение и закрывает приложение.

## Установка и запуск

```bash
open build/Hopkey.app          # просто запустить
cp -R build/Hopkey.app /Applications/   # установить
```

> **Снятие карантина (обязательно для скачанного релиза).** Приложение подписано
> self-signed-сертификатом без нотаризации Apple, поэтому скачанный из интернета
> бандл (DMG из GitHub Releases) macOS блокирует с ошибкой «приложение
> повреждено». Разрешите запуск одной командой:
>
> ```bash
> xattr -dr com.apple.quarantine /Applications/Hopkey.app
> ```
>
> Для локально собранного `build/Hopkey.app` это не требуется (карантин ставится
> только на скачанные файлы), но команда безопасна и в этом случае.

Иконка-«билетик» появится в строке меню. Из меню:

- **Открыть ссылку из буфера** — разобрать текущий буфер и открыть ссылку в браузере.
- **Скопировать ссылку из буфера** — собрать ссылку из буфера и положить её в буфер.
- **Открыть по ключу…** — окно ручного ввода ключа/номера.
- **Настройки…** — шаблоны, хоткеи, общие параметры и язык интерфейса.
- **Проверить обновления…** — проверка обновлений (Sparkle).
- **Выход**.

Переключатели (действие при копировании, хоткеи, запуск при входе, автообновление,
язык) живут в окне **«Настройки»**, а не в меню.

## Разрешения

- **Уведомления** — запрашиваются при первом запуске (для кликабельного баннера). Если не выдать,
  приложение в режиме «уведомлять» откроет ссылку напрямую.
- **Accessibility** — нужен только если включаете глобальный хоткей (для синтеза `Cmd+C` и чтения
  выделения). Системные настройки → Конфиденциальность и безопасность → Универсальный доступ.

## Настройка

При первом запуске окно настроек открывается автоматически. Распознавание задаётся **шаблонами**
(regex → URL) — каждый шаблон это:

- **Шаблон (regex)** — что искать в тексте, например `PROJ-(\d+)` или `#(\d+)`.
- **URL** — куда открывать; группы совпадения подставляются как `$1` (первая группа), `$2`, …,
  `$0` — всё совпадение. Например `https://your-jira/browse/PROJ-$1`.
- **Слово** — требовать границы слова (чтобы `XPROJ-1` не ловился). **ВЕРХ** — нормализовать в
  верхний регистр (нужно Jira-ключам). **Вкл** — участвует ли шаблон в распознавании.

Кнопка **«Из пресета»** добавляет готовую заготовку (Jira, GitHub, GitLab, CVE) — остаётся вписать
свой домен/репозиторий. Так Jira — просто один из шаблонов, а Hopkey открывает любые ID.

Хранится в `UserDefaults`. Прежние «проекты Jira» автоматически переносятся в шаблоны при первом
запуске новой версии — по одному шаблону на каждый префикс. Пока нет ни одного валидного шаблона,
распознавание не работает.

## Язык интерфейса

Интерфейс доступен на **русском и английском**. По умолчанию язык выбирается по
системным настройкам (если ни один не подходит — английский).

Переключить вручную можно в **Настройки → Общие → «Язык интерфейса»**: «Системный»,
«Русский» или «English». Смена применяется после перезапуска приложения (приложение
само предложит перезапуститься). Альтернатива — системный путь: Системные настройки →
Основные → Язык и регион → раздел «Программы».

Строки хранятся в `Sources/Hopkey/Resources/{en,ru}.lproj/Localizable.strings` и
подставляются через хелпер `L(_:)`. Как добавлять/менять строки — см. [AGENTS.md](AGENTS.md).
